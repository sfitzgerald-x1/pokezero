The best-controlled view we have. Trait ↔ foul-play win-rate correlation

Pearson r across 3 checkpoints. Both the trait and the win rate are measured on the same foul-play games (v3-k16@100k 0.11, v3-k32@100k 0.14, v3-k64@100k 0.14). Green = trait tracks a higher foul-play win rate, red = lower.

One point per checkpoint, not per game: each checkpoint's 1000 foul-play games collapse into a single (trait, win-rate) pair, so n is the number of checkpoints — game volume buys precision within a point, not more points. It is also an aggregate correlation confounded by overall checkpoint strength (better models do more of everything effective and win more), so read it as association, not cause.

Low power: n=3 checkpoints over a 0.035 win-rate spread. With this little variance the r values are unstable — treat as hypothesis-generating only. Foul-play on more checkpoints widens the win-rate axis and fixes this.
