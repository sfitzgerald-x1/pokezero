The best-controlled view we have. Both seats are the same policy playing the same game, so comparing the winning seat's behavior against the losing seat's holds policy strength and game length fixed by construction — a game-level quantity has no within-game variance and cannot leak in. 84 checkpoints, 349,896 decided seat-games total. Trait ↔ foul-play win-rate correlation

Pearson r across 6 checkpoints. Both the trait and the win rate are measured on the same foul-play games (m50-ep7@500k 0.20, m50-ep7@1000k 0.35, l200-ep7-wu75@500k 0.32, l200-ep7-wu75@800k 0.30, v22-lr3m@500k 0.31, v22-lr3m@1900k 0.42). Green = trait tracks a higher foul-play win rate, red = lower.

One point per checkpoint, not per game: each checkpoint's 1000 foul-play games collapse into a single (trait, win-rate) pair, so n is the number of checkpoints — game volume buys precision within a point, not more points. It is also an aggregate correlation confounded by overall checkpoint strength (better models do more of everything effective and win more), so read it as association, not cause.

Low power: n=6 checkpoints over a 0.222 win-rate spread. With this little variance the r values are unstable — treat as hypothesis-generating only. Foul-play on more checkpoints widens the win-rate axis and fixes this.
