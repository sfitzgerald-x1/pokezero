1. Overview & goal

PokeZero trains a model to play Pokémon Showdown Gen 3 random battles through repeated self-play, using AlphaGo Zero as conceptual inspiration — the self-play learning loop, a shared policy/value network, and learning from game outcomes with no hand-coded strategy.

The current milestone is deliberately CPU-only. The active goal for this report was concrete and measurable:

Goal. Produce a CPU-trained policy that meaningfully beats the simple-legal strategy, demonstrated with reproducible shared-opponent benchmark evidence — Wilson 95% lower bound ≥ 0.55 over ≥ 240 mirrored games, reproduced across two independent seed bands, with clean capped-game health.

3. System design: environment & observations

The simulator owns full battle state (including hidden teams, moves, items, abilities). The policy only ever sees an information state a real player could know.

  • Player-relative normalization. Every observation is framed as self_* vs opponent_* regardless of the raw Showdown p1/p2 seat, so the same position observed by either side yields mirrored inputs. This is mandatory for self-play and two-bot evaluation.
  • Fixed 9-action space. Slots 0–3 are active moves, 4–8 are switches in canonical team order; an explicit legal-action mask gates illegal/disabled/trapped choices.
  • Belief tracking. A deterministic, source-backed Gen 3 randbat belief engine narrows the candidate sets for each revealed opponent slot from public events. Compact belief features (possible abilities/items/moves, counts, uncertainty) are exposed in the observation.
  • Rollout format. Trajectories are recorded as JSONL with observations, legal masks, chosen actions, opponent-action labels, terminal outcomes, capped-game markers, and policy identifiers. Games end normally or at a provisional 250-turn cap.
Why random battles are a tractable target. Unlike human team-built formats, every Pokémon is drawn from a known generator with a small, enumerable set of possible moves/items/abilities. Opponent uncertainty is therefore a posterior over a finite, listable space — which makes belief modeling, opponent prediction, and any future search far cheaper.

4. Policy classes: linear vs. transformer

Linear baseline (scaffolding)

A dependency-free, CPU-only masked linear softmax policy over hashed features, trainable by behavior cloning or reward-weighting. It validated the full harness — environment boundary, trajectory format, checkpoints, self-play loop, promotion gates, benchmarking — but cannot model interactions between board state, the active matchup, and a candidate move. As shown below, it plateaus at parity with simple-legal.

Entity-token transformer (the model that cleared the goal)

A small PyTorch transformer over entity / action / history tokens, behind the optional [neural] extra. It embeds categorical ids (species, moves, items, abilities, statuses, weather), numeric features (HP, boosts, turn progress, set counts, uncertainty), token types, and history position; a transformer encoder feeds three heads: policy (9 action logits), value (scalar), and an auxiliary opponent-action prediction head over a shared trunk.

5. Learning architecture direction

AlphaGo Zero does not transfer literally: Pokémon is imperfect-information (hidden teams), stochastic (damage rolls, accuracy, crits, secondaries), uses simultaneous moves, and has no cheap perfect forward model (the simulator is a subprocess). So we keep the self-play loop and value head, and treat search as an optional later accelerator. The chosen direction:

  • Operator: model-free self-play RL (PPO / V-trace) first; AlphaZero-style search (determinized / information-set MCTS) only later, and only if a fast forward model exists and search is shown to help.
  • Network: per-turn set-encoder over entities → recurrent core across turns → policy / value / opponent-action / opponent-belief heads.
  • Temporal opponent inference: history must inform hidden-card prediction (e.g. an opponent repeatedly switching out of a matchup implies it can't threaten you → predict the switch and punish it).
  • Privileged self-play supervision: because self-play gives ground truth for both teams, belief/opponent heads can be supervised directly at training time while the policy stays public-information-only at inference.
  • Open tension (to be settled by ablation, not assumption): whether an explicit belief head is needed or whether opponent inference emerges in a recurrent state — treated as a tunable loss weight that can anneal to zero.
  • Compute: CPU proves the design is correct (this report); GPU plus environment throughput is what buys strength. At proof-of-concept scale the workload is environment-bound, not compute-bound.
  • Observability: TensorBoard for now (local, free live curves); a custom dashboard deferred until Pokémon-specific views (battle replay, belief-probe visualization) are needed.

6. Autonomous development loop

Work is produced by an inverted, cross-model loop so that review is genuinely independent:

  • Executor — Claude Code. Selects tasks, implements, tests, opens PRs, applies review findings, merges per policy. Runs from an isolated git worktree to avoid working-tree collisions.
  • Reviewer — Codex (GPT-5.5, xhigh reasoning). Adversarial, read-only review of every code PR; cannot modify files or merge (enforced by a read-only sandbox).
  • Merge policy. Serial; under an explicit goal, a code PR merges once it passes local verification and Codex signs off with no blocking findings. Documentation-only PRs merge without a Codex review.

This loop was validated end-to-end during this milestone: the one code change (declaring numpy in the [neural] extra) was reviewed by Codex — "no blocking issues found" — and merged automatically.

8. Caveats & next steps

This was the cheapest credible neural attempt, so scope is bounded:
  • It is supervised behavior-cloning + value, not the self-play RL loop of full Stage 1.
  • Strength is measured only against random-legal / simple-legal, not against scripted-teacher or the linear reference head-to-head.
  • Band-to-band variance is real (0.81 vs 0.73); larger samples are warranted before treating it as a tuned incumbent.

Natural next milestones: (1) benchmark the transformer vs scripted-teacher and the linear reference directly; (2) scale data/model and re-measure; (3) build the actual model-free RL Stage 1 loop with the temporal opponent-inference network and privileged belief supervision.
